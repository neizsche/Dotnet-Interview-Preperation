# .NET Interview Study Guide

This repo is organized to reduce topic overlap during daily prep while keeping all original notes.
Each primary note now has a study-first structure:
- Focus
- Topic Map
- Study Notes (Simplified)
- Rapid Interview Q&A
- Preserved detailed notes

## Folder Layout

- `notes/`  
  Main study path. Start here for interview preparation.
  Main tracks use broader topic names, and some tracks now combine related subdomains.
  Security and testing subtopics are grouped under `notes/dotnet&backend/`.
  Detailed files keep `NN-XX-...` naming for consistent ordering (for example, `01-00-csharp.md`, `01-01-...`).
- `checklists/`  
  Quick revision and coverage tracking lists.

## Recommended Study Order

1. `notes/c#/README.md`
2. `notes/dotnet&backend/README.md`
3. `notes/architecture/README.md`
4. `notes/database/README.md`
5. `notes/scrum/README.md`
6. `notes/angular/README.md` (optional for full-stack roles)
