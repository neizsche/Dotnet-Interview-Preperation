# Interview Checklists

Use these lists to verify topic coverage before interviews.

## Files

1. `csharp-topics-checklist.md`
2. `dotnet-topics-checklist.md`
3. `arch-topics-checklist.md`
4. `sql-topics-checklist.md`
5. `interview-playbook-prompt-template.md`

## Ownership Split

1. `csharp-topics-checklist.md` owns language/runtime topics.
2. `dotnet-topics-checklist.md` owns framework, backend, architecture, and operational topics.
3. `arch-topics-checklist.md` owns design patterns and advanced architecture deep-dive topics.
4. `sql-topics-checklist.md` owns database and SQL topics.
5. Cross-references are used instead of duplicating checklist items across files.
6. `interview-playbook-prompt-template.md` stores the reusable transformation prompt for converting raw notes into senior interview playbooks.

## How to Use

1. Mark completed topics after each study session.
2. Revisit weak topics in `notes/`.
3. Pull deeper references from nearby `NN-XX` subtopic files.
